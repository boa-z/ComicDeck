## Documentation Rules

- Keep sections self-contained and scannable.
- Prefer concrete paths, screen names, and commands over vague descriptions.
- Update affected docs in the same change as product or architecture work.
- Record deliberate product constraints explicitly so they do not get lost in implementation churn.
